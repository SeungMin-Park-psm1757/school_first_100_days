# PPT 15장 완성본

## 1. 표지
- 제목: 우리 아이 첫 100일
- 부제: 초등 1학년 학부모 지원 네비게이터
- 설명: 교육 공공데이터와 AI를 결합한 시기별 의사결정 지원 플랫폼

## 2. 문제 정의
- 초등학교 입학 초기에는 준비물, 알림장, 하교 동선, 방과후, 돌봄, 생활 루틴이 동시에 몰립니다.
- 학부모는 많은 안내를 받아도 “지금 무엇을 먼저 해야 하는가”를 판단하기 어렵습니다.
- 필요한 것은 단순 정보 제공이 아니라 시점별 행동 우선순위를 정리해 주는 시스템입니다.

## 3. 왜 하필 첫 100일인가
- 초등학교 1학년 입학 초기에는 학교생활 적응과 생활 루틴 형성이 동시에 일어납니다.
- 이 시기를 별도 사용자 여정으로 재구성하면 문제와 해결안이 훨씬 선명해집니다.

## 4. 기존 방식의 한계
- 공식 정보는 학교알리미, 교육청 자료, 학교 안내문, 공공데이터 포털에 흩어져 있습니다.
- 선배 학부모 경험담은 유용하지만 구조화되어 있지 않습니다.
- 결과적으로 정보 조회는 되지만 행동 안내는 약합니다.

## 5. 해결안
- 입학 전후 100일 학부모 지원 네비게이터
- 학교, 지역, 보호자 일정, 자녀 특성을 입력하면 AI가 이번 주 해야 할 일과 주의 포인트를 제시합니다.

## 6. 대상 사용자
- 초등학교 입학이 확정된 초등 1학년 학부모
- 맞벌이/돌봄 공백/입학 초기 적응 부담이 큰 가정을 우선 시나리오로 설계

## 7. 서비스 구조
- 1층: 학교알리미와 교육부 공공데이터를 활용한 공식 정보 레이어
- 2층: 입학 준비-적응-루틴 정착-돌봄 재조정-100일 회고 타임라인
- 3층: 선배 학부모 경험 레이어

## 8. 핵심 기능
- 100일 타임라인 기반 체크리스트
- 방과후/돌봄 시기별 의사결정 지원
- AI 행동 가이드
- 후기 요약 및 전수

## 9. AI 역할
- 챗봇이 아니라 행동 가이드 생성기
- 해야 할 일, 주의사항, 추천 선택지, 후기 요약을 JSON 형태로 생성

## 10. 데이터 연결
- 학교알리미 공시정보
- 방과후학교 운영 계획 및 운영 지원 현황
- 초등돌봄(온종일돌봄) 관련 현황
- 공개 데이터는 “무엇이 있는가”를 보여주고, 서비스는 “지금 무엇을 해야 하는가”를 안내합니다.

## 11. 화면 1: 홈/온보딩
- 학교명, 거주지, 보호자 근무 형태, 자녀 특성, 주요 고민 입력
- AI가 즉시 “이번 주 해야 할 일 3가지” 제시

## 12. 화면 2: 100일 타임라인
- 입학 준비 / 첫 2주 / 1개월 적응 / 돌봄 재조정 / 100일 회고
- 카드형 행동 가이드와 완료 체크

## 13. 화면 3: 추천 결과
- 방과후·돌봄 추천 카드
- 추천 이유, 확인 필요 사항, 선배 팁 요약 표시

## 14. MVP 구현안
- Next.js + TypeScript + Tailwind CSS
- JSON 더미 데이터
- 로그인/DB/실시간 연동 제외
- 데모용 API 라우트로 AI 결과 시연

## 15. 기대 효과 및 경쟁력
- 초등 입학 초기라는 선명한 문제 구간
- 교육 공공데이터를 단순 조회가 아닌 의사결정 도구로 재구성
- 웹 데모까지 보여줘 실현 가능성 확보
